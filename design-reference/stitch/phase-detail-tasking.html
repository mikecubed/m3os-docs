biotech Implementation Deep Dive

Context switching in m³OS is performed by saving the current CPU state onto the task's stack and restoring the state of the next task. In Rust, we use naked functions to ensure the compiler doesn't insert any prologue or epilogue that could corrupt the stack during the switch.

The core routine involves swapping the rsp register and utilizing the `pushq`/`popq` instructions to preserve callee-saved registers (rbx, rbp, r12-r15).

kernel/task.rs content_copy
01020304050607080910
// Perform context switch between two tasks
#[naked]
pub unsafe extern "C" fn switch(
_old_stack: *mut usize,
_new_stack: *const usize
) {
// ASM routine to swap RSP and registers
asm!("push rbp", "push rbx", options(noreturn));
}

lightbulb Design Decision

Cooperative vs Preemptive

m³OS utilizes preemptive scheduling powered by the APIC timer. This ensures no single task can starve the system, providing high responsiveness for kernel-level services.

m³OS Scheduler

Simple Round-Robin. Each task gets a fixed 10ms slice.

Linux Kernel

Completely Fair Scheduler (CFS). Uses red-black trees to balance CPU time based on 'vruntime'.
