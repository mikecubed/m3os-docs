void sched_yield() {
  // Lock runqueue
  spin_lock(&rq->lock);
  
  struct task *curr = rq->curr;
  curr->state = TASK_READY;
  enqueue_task(rq, curr);
  
  pick_next_task(rq);
  spin_unlock(&rq->lock);
}

Unlike Linux's fair_sched, m³OS enforces hard realtime deadlines for IRQ-driven tasks, preventing user-space starvation during high-I/O bursts.
