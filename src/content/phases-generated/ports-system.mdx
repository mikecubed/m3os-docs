## Why this phase matters

A simple "ports" system enables building and installing third-party software inside the OS from source recipes. A port is a directory containing a `Makefile`, patches, and a description file that automates downloading (or bundling), compiling, and installing a piece of software.

## How it works in m3OS

The roadmap entry defines the learning scope for this phase; a deeper implementation doc has not been matched yet.

## How it builds on earlier phases

This phase builds on Rust Cross Compilation and turns that foundation into Ports System.

## Implementation outline

- Design the `Portfile` format (keep it simple: shell variables).
- Write the `port` script/program.
- Create the ports tree directory structure in the disk image.
- Port Lua as the first test case (small, portable, no dependencies).
- Verify: `port install lua` builds and installs Lua, `lua -e "print('hello')"` works.
- Port remaining initial packages, adding patches as needed.
- Implement dependency resolution in the `port` command.
- Track installed packages.
- Document how to create a new port.
