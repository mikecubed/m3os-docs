## Why this phase matters

Run a C compiler natively inside the OS. A C source file written at the shell prompt (using the editor from Phase 26) can be compiled and executed without leaving the OS. The ultimate target is self-hosting: the compiler compiles itself.

## How it works in m3OS

Phase 31 adds a native C compiler (TCC -- Tiny C Compiler) to m3OS. A C source file written with the text editor can be compiled and executed without leaving the OS. The ultimate milestone is self-hosting: TCC compiles itself inside the OS, producing a working compiler binary.

## How it builds on earlier phases

This phase builds on Telnet Server and turns that foundation into Compiler Bootstrap.

## Implementation outline

- On the host: build TCC from source with `./configure --prefix=/usr --cc=x86_64-linux-musl-gcc`.
- Verify the resulting binary is a static x86-64 ELF linked against musl.
- Add TCC binary, musl `libc.a`, and the musl headers to the disk image build in xtask.
- Boot the OS and verify `tcc --version` prints the expected string.
- Write a `hello.c` inside the OS using the editor, compile it, and run it.
- Compile a slightly larger program (e.g., a fibonacci calculator) to stress the heap.
- Attempt the self-hosting milestone: `tcc /usr/src/tcc/tcc.c -o /tmp/tcc2`.
- Verify the self-compiled TCC produces identical output for `hello.c`.
