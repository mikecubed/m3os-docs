## Why this phase matters

The OS supports PS/2 mouse input. Userspace programs can read mouse movement deltas and button state via a device file or syscall. The mouse is usable in the text editor for cursor positioning and in graphical programs like DOOM for aiming.

## How it builds on earlier phases

This phase builds on Doom and turns that foundation into Mouse Input.

## Implementation outline

- Initialize PS/2 auxiliary port and enable mouse reporting.
- Add IRQ 12 handler that accumulates 3-byte packets.
- Add mouse packet ring buffer (similar to keyboard scancode buffer).
- Implement userspace interface (device file or syscall).
- Write a mouse test program that prints movement and button events.
- Integrate with DOOM platform layer for mouse aiming.
- Optionally add software cursor for framebuffer mode.
