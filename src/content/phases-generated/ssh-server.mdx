## Why this phase matters

An SSH server runs inside the OS, providing encrypted remote shell access. Users can connect from any standard SSH client (`ssh user@host`) with password or public key authentication. This replaces the telnet server for untrusted networks and is a major milestone: the OS is now a secure, networked, multi-user system.

## Implementation outline

- Cross-compile Dropbear with musl (or begin SSH protocol implementation).
- Generate Ed25519 host key at first boot.
- Implement SSH transport: version exchange, key exchange, encrypted packets.
- Implement password authentication against `/etc/shadow`.
- Implement session channel with PTY allocation.
- Implement shell execution (fork + setsid + exec login).
- Test from host with `ssh -p 2222 user@localhost`.
- Implement public key authentication.
- Add sshd to init's startup sequence.
- Test multiple simultaneous SSH sessions.
