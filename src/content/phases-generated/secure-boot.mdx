## Why this phase matters

Sign the kernel's EFI binary so it can boot on real hardware with UEFI Secure Boot enabled, without needing to disable the firmware security check.

## Implementation outline

- Generate a 4096-bit RSA key pair and self-signed X.509 certificate with `openssl`.
- Add `cargo xtask sign <unsigned-efi>` for signing any EFI executable with optional `--key` / `--cert` overrides.
- Extend `cargo xtask image` with `--sign` so the build produces a signed bootable disk image in addition to the unsigned image.
- Document the one-time certificate enrollment. Two paths exist — choose based on the target machine (see below).
- Verify the signed binary boots with `mokutil --sb-state` or `dmesg` reporting Secure Boot enabled.
- Put the firmware into **Setup Mode** (clear the Platform Key via UEFI setup).
- Enroll your own PK, KEK, and db certificates using `efi-updatevar` or the firmware setup UI.
- The firmware then trusts your signed binary directly, without shim.

- Measured boot / TPM attestation (verifying the kernel hash against a TPM PCR).
