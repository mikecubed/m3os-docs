## Why this phase matters

Implement real pseudo-terminal (PTY) pairs so that multiple independent terminal sessions can run simultaneously. A PTY master/slave pair connects a terminal emulator (or network server) to a shell process, enabling remote access (telnet, SSH) and local terminal multiplexing.

## How it works in m3OS

Phase 29 implements pseudo-terminal (PTY) pairs, enabling multiple independent terminal sessions. A PTY master/slave pair connects a terminal emulator (or network server) to a shell process — this is the mechanism behind SSH, screen, tmux, and xterm.
The implementation provides a fixed pool of 16 PTY pairs with bidirectional ring buffers, per-PTY terminal settings, line discipline processing on the slave side, session management (`setsid`/`getsid`), and controlling terminal association (`TIOCSCTTY`/`TIOCNOTTY`).

## How it builds on earlier phases

This phase builds on Ext2 Filesystem and turns that foundation into PTY Subsystem.

## Implementation outline

- Implement the PTY allocator: a table of PTY pairs, each with a ring buffer connecting master and slave.
- Implement `/dev/ptmx` as a special device that allocates a new PTY pair.
- Connect the slave side to the termios line discipline from Phase 22.
- Implement `setsid()` syscall for creating new sessions.
- Implement `TIOCSCTTY` ioctl to assign a controlling terminal.
- Modify `fork`/`exec` in login/shell to open a PTY slave as stdin/stdout/stderr for new sessions.
- Write a simple test program that spawns a shell on a PTY and relays I/O.
- Verify that terminal features (raw mode, ANSI escapes, signals) work through PTYs.
