## Why this phase matters

Load and execute arbitrary userspace binaries from disk as isolated ring-3 processes, with a proper lifecycle: spawn, run, wait, exit.

## How it builds on earlier phases

## Implementation outline

- Write an ELF64 parser that reads a file from the VFS and validates the header.
- Allocate a fresh page table root and map each `PT_LOAD` segment at the right virtual address.
- Allocate and map a userspace stack; push `argc`/`argv`/`envp` in the System V ABI layout.
- Wire `execve` to load a new image into the calling process's address space.
- Wire `fork` to copy the parent's page tables and kernel state into a new process entry.
- Wire `exit` to mark the process as zombie and wake any waiting parent.
- Wire `waitpid` to block until a matching child transitions to zombie, then reap it.
