## Why this phase matters

The OS supports `AF_UNIX` (Unix domain) sockets for high-performance local inter-process communication. Programs can create stream and datagram sockets bound to filesystem paths or abstract addresses. This is the primary IPC mechanism on Linux, used by D-Bus, X11, systemd, PostgreSQL, Docker, and many other systems.

## Implementation outline

- Add `AF_UNIX` socket type to the socket syscall dispatcher.
- Implement `UnixSocket` kernel data structure with ring buffers.
- Implement `socketpair()` — simplest case (no filesystem, pre-connected).
- Implement `bind()` for named sockets (create socket file in VFS).
- Implement `connect()` to named sockets (find socket file, establish connection).
- Implement `listen()` + `accept()` for stream sockets.
- Implement `read()`/`write()` for connected stream sockets.
- Implement `sendto()`/`recvfrom()` for datagram sockets.
- Add poll/epoll support for Unix socket fds.
- Test: syslog-style server accepting connections on `/dev/log`.
- Test: `socketpair()` for parent-child communication.
