## Why this phase matters

The OS supports `AF_UNIX` (Unix domain) sockets for high-performance local inter-process communication. Programs can create stream and datagram sockets bound to filesystem paths. `socketpair()` creates connected pairs for parent-child IPC after `fork()`. This is the primary IPC mechanism on Linux, used by D-Bus, X11, systemd, PostgreSQL, Docker, and many other systems.

## Implementation outline

- Add `UnixSocket` struct and `UNIX_SOCKET_TABLE` in `kernel/src/net/unix.rs`.
- Add `FdBackend::UnixSocket` variant to process FD infrastructure.
- Extend `sys_socket()` to accept `AF_UNIX` and allocate Unix sockets.
- Implement `sys_socketpair()` with real Unix socket pairs (replace pipe delegation).
- Implement `bind()` for named sockets (create socket file in VFS).
- Implement `connect()` to named sockets (find socket file, establish connection).
- Implement `listen()` + `accept()` for stream sockets.
- Implement `read()`/`write()` for connected stream sockets.
- Implement `sendto()`/`recvfrom()` for datagram sockets.
- Add poll/epoll support for Unix socket fds.
- Add `shutdown()` support for Unix sockets.
- Test: `socketpair()` for parent-child communication.
- Test: named stream socket server/client.
- Test: datagram socket message boundary preservation.
