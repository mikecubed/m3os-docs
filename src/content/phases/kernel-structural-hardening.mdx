## Why this phase matters

The kernel's memory management, context management, and TLB coherence subsystems move onto the structurally hardened path defined by this phase. In the checked-in tree, the AddressSpace object, typed user buffers, batched TLB work, and frame zeroing landed, and Phase 52d completed the remaining closure work so the task-owned return-state and generation-tracking story now matches the shipped implementation end to end.

## How it works in m3OS

Phase 52b replaces the fragile kernel patterns that made the Phase 52 bugs possible. Where Phase 52a patched the immediate symptoms, this phase introduces structural defenses: a first-class `AddressSpace` object with per-core tracking, typed `UserSliceRo`/`UserSliceWo` wrappers at the syscall boundary, task-owned `UserReturnState` that eliminates ~40 manual restore call sites, batch TLB shootdown with address-space targeting, and frame zeroing on free. These patterns are drawn from Redox, seL4, and Zircon architecture analysis.

## How it builds on earlier phases

This phase builds on Kernel Reliability Fixes and turns that foundation into Kernel Structural Hardening.

## Implementation outline

- Define `AddressSpace` struct, wrap existing `page_table_root` + VMAs
- Update `Process` to use `Arc<AddressSpace>` instead of `Option<PhysAddr>`
- Update scheduler dispatch to call `activate_on_core`/`deactivate_on_core`
- Add `current_addrspace` pointer to `PerCoreData`
- Implement batch `tlb_shootdown_range` using `active_on_cores`
- Define `UserReturnState`, add to `Task`
- Save user state to `UserReturnState` at syscall entry
- Restore from `UserReturnState` in scheduler dispatch
- Remove all `restore_caller_context` calls
- Define `UserSliceRo`/`UserSliceWo`/`UserSliceRw`, convert syscall handlers
- Add frame zeroing to `free_frame`
- Add generation counter assertions around `copy_to_user`
