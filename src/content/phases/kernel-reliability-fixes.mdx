## Why this phase matters

All four confirmed kernel bugs from the Phase 52 bug investigations are fixed, with targeted regression tests proving each fix.

## How it works in m3OS

Phase 52a addresses four confirmed kernel bugs discovered during the Phase 52 service extraction work. These are not design improvements -- they are direct correctness fixes for stale IPC return state, a vendored SSH library waker bug, missing POSIX `clear_child_tid` on thread exit, and missing signal action reset on exec. All four bugs blocked either the `copy_to_user` reliability investigation or the SSHD post-authentication hang.

## How it builds on earlier phases

This phase builds on First Service Extractions and turns that foundation into Kernel Reliability Fixes.

## Implementation outline

- Add `restore_caller_context` to `kernel/src/ipc/mod.rs` dispatch function
- Fix `Channel::wake_write()` in `sunset-local/src/channel.rs`
- Add `clear_child_tid` cleanup to thread exit in `kernel/src/arch/x86_64/syscall/mod.rs`
- Add signal action reset to `sys_execve` in `kernel/src/arch/x86_64/syscall/mod.rs`
- Write targeted regression tests for each fix
