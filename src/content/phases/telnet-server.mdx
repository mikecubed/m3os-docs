## Why this phase matters

A telnet server runs inside the OS, allowing remote users to log in over the network and get an interactive shell session. This is the first demonstration of the OS as a networked, multi-user system — a machine you can connect to from another computer (or another terminal on the host) and do real work.

## How it works in m3OS

Phase 30 adds `telnetd`, a telnet server that listens on TCP port 23, accepts remote connections, allocates a PTY pair per session, and relays data between the TCP socket and the terminal.  This is the first demonstration of m3OS as a networked, multi-user system -- a machine you can log into from another computer.

## How it builds on earlier phases

This phase builds on PTY Subsystem and turns that foundation into Telnet Server.

## Implementation outline

- Write a minimal telnet server in C (cross-compiled with musl).
- Implement IAC parsing: strip telnet commands from the data stream.
- Implement option negotiation for ECHO, SGA, and NAWS.
- On accept: fork, allocate PTY, exec login on the slave side.
- Main loop: `poll()` on both the socket fd and the PTY master fd, relay data.
- Handle connection close: send `SIGHUP` to the session, clean up.
- Add telnetd to init's startup sequence.
- Configure QEMU with port forwarding: `-nic user,hostfwd=tcp::2323-:23`.
- Test from host: `telnet localhost 2323`.
