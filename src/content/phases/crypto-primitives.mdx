## Why this phase matters

The OS has a cryptography library providing hash functions, symmetric encryption, and asymmetric key operations. This is the foundation layer that SSH (Phase 43) and future security features build upon.

## How it works in m3OS

The roadmap entry defines the learning scope for this phase; a deeper implementation doc has not been matched yet.

## How it builds on earlier phases

This phase builds on Expanded Coreutils and turns that foundation into Crypto Primitives.

## Implementation outline

- Add RustCrypto crate dependencies to the workspace (`sha2`, `hmac`, `hkdf`, `chacha20poly1305`, `aes`, `ctr`, `ed25519-dalek`, `x25519-dalek`, `rand_chacha`).
- Create `userspace/crypto-lib/` crate wrapping the RustCrypto APIs.
- Implement CSPRNG seeded from `getrandom` syscall.
- Verify SHA-256 produces correct test vectors inside the OS.
- Verify HMAC-SHA-256 and HKDF with test vectors.
- Verify ChaCha20-Poly1305 and AES-256 with test vectors.
- Verify Ed25519 sign/verify with test vectors.
- Verify X25519 key exchange with test vectors.
- Build `sha256sum` utility.
- Build `genkey` utility.
- Run full integration test inside QEMU.
- Document the library's API for use by the SSH server in Phase 43.
