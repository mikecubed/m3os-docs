## Why this phase matters

Tasking is the moment the OS stops being a single control flow and becomes a system. Once execution can move between tasks, later concepts like servers, IPC, and userspace processes become much easier to explain.

## How it works in m3OS

The scheduler keeps a compact set of runnable tasks and rotates through them in round-robin order. The switch stub saves the callee-saved register frame and flags, swaps stacks, and restores the next task so execution can resume as if it had simply returned from a function call.

## How it builds on earlier phases

Tasking depends on memory management for stacks and on interrupts for timer signaling. It extends the boot and diagnostic foundation by making execution state more complex while still keeping behavior teachable.
