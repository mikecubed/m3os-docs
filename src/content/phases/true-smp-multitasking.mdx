## Why this phase matters

All CPU cores dispatch and run user tasks. The BSP-only restriction is removed by making syscall infrastructure per-core. The scheduler introduces per-CPU run queues with load balancing and task priorities, substantially improving dispatch locality and multi-core throughput even though the global scheduler lock remains on the dispatch hot path for task-state coordination.

## Implementation outline

- Move syscall statics into `PerCoreData` and update `syscall_entry` assembly.
- Verify: two cores can handle syscalls simultaneously without corruption.
- Remove BSP-only guard in `pick_next()`.
- Verify: tasks run on multiple cores (log which core runs which task).
- Implement per-CPU run queues and move ready-queue contention off the single global queue while keeping the global scheduler lock for task-state coordination.
- Implement load balancing (periodic migration).
- Add priority field to `Task`; update `pick_next()` to respect priorities.
- Implement `sched_setaffinity` / `sched_getaffinity` syscalls.
- Implement wait queues; attach to pipes and sockets.
- Add time accounting; implement `times()` syscall.
- Stress test with many concurrent processes across all cores.
