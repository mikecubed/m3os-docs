## Why this phase matters

Rust programs compiled on the host run natively inside the OS. A documented toolchain setup enables writing Rust code on the host, cross-compiling to an x86-64 ELF binary linked against musl, and deploying it to the OS's disk image. Stretch goal: a minimal Rust program compiles inside the OS itself.

## Implementation outline

- Set up `x86_64-unknown-linux-musl` target on the host.
- Write and cross-compile `hello-rust` — verify it runs in the OS.
- Identify and implement any missing syscalls that `std` needs.
- Write the demonstration programs.
- Extend xtask to cross-compile and package Rust userspace programs.
- Create the custom m3os target specification for `no_std` programs.
- Write a `syscall` crate for native m3os programs.
- Document the full cross-compilation workflow.
- (Stretch) Investigate porting mrustc or rustc.
