## Why this phase matters

A usable text editor runs inside the OS. Users can create, edit, and save files from the shell. This is the foundational tool that makes all subsequent "do real work inside the OS" phases possible — you cannot write code, configuration files, or documents without an editor.

## How it works in m3OS

Phase 26 adds `edit`, a minimal full-screen text editor for m3OS. It is a no_std Rust binary based on kibi/kilo concepts: a line-based text buffer, VT100 escape sequences for screen control, raw terminal mode, and a single-file editing model.
The editor is available at `/bin/edit` and is set as the `$EDITOR` environment variable.

## How it builds on earlier phases

This phase builds on SMP and turns that foundation into Text Editor.

## Implementation outline

- Extend `syscall-lib` with ioctl, lseek, termios, and winsize wrappers.
- Add a userspace heap allocator (kibi uses `Vec`/`String` via `alloc`).
- Create `userspace/edit/` crate (no_std, `x86_64-unknown-none` target).
- Write an m3OS platform backend (~60-80 lines) implementing terminal I/O.
- Port kibi's core modules, replacing `std::fs`/`std::io` with syscall-lib.
- Verify raw mode, file save/load, and search work end-to-end.
- Add to xtask build, initrd, and set `$EDITOR=/bin/edit`.
