## Why this phase matters

The OS supports kernel-level threads via `clone()` with `CLONE_THREAD` and `CLONE_VM`. Multiple threads share an address space and can synchronize via `futex()`. Thread-local storage works via `arch_prctl(ARCH_SET_FS)`. This enables multi-threaded C programs and improves musl libc compatibility (which uses threads internally even for single-threaded programs).

## Implementation outline

- Add `tid`, `tgid`, and `clear_child_tid` fields to `Process`; implement `gettid()`.
- Implement `ThreadGroup` struct and membership tracking.
- Extend `clone()` to accept `CLONE_VM | CLONE_THREAD | CLONE_FILES | CLONE_SIGHAND`.
- Allocate per-thread kernel stacks; share page table root with `CLONE_VM`.
- Implement `CLONE_SETTLS` (set FS base for new thread).
- Implement `CLONE_CHILD_CLEARTID` and `set_tid_address()`.
- Implement `FUTEX_WAIT` and `FUTEX_WAKE` with proper wait queues.
- Implement `FUTEX_WAIT_BITSET` and `FUTEX_WAKE_BITSET`.
- Update `exit()` for thread-only exit vs `exit_group()` for group exit.
- Update signal delivery for thread groups (`tkill`, process-directed signals).
- Build userspace thread test program.
- Validate with musl `pthread_create` / `pthread_join` if possible.
