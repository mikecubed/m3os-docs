## Why this phase matters

Multi-file C projects can be built inside the OS using a `make`-compatible build tool and shell scripts. This transforms the OS from a "compile one file" environment into one where real software projects can be developed.

## How it works in m3OS

Phase 32 adds a POSIX-compatible `make` build tool and supporting utilities, enabling multi-file C projects to be built inside the OS. This transforms the OS from a "compile one file" environment into one where real software projects can be developed with incremental builds.

## How it builds on earlier phases

This phase builds on Compiler Bootstrap and turns that foundation into Build Tools.

## Implementation outline

- Cross-compile pdpmake (or chosen make) with musl.
- Add `make` binary to disk image at `/usr/bin/make`.
- Verify basic Makefile parsing and rule execution.
- Cross-compile `ar`, `install`, `touch`, `wc`, `stat` utilities.
- Add them to the disk image.
- Create the demo multi-file project in the disk image.
- Boot, run `make` in the project directory, verify incremental builds work.
- Test shell scripting: write a `build.sh` that automates a build-and-test cycle.
