### Why route file access through IPC?
The same policy/mechanism argument from Phase 7 applies here. The kernel should own only the mechanism — endpoint delivery, scheduling, memory protection. Policy questions (which process can read which path, how to handle missing files, what backends exist) belong in userspace servers.
Routing through `vfs_server` instead of letting clients call `fat_server` directly also creates the ownership boundary needed for Phase 9+: adding a second backend (tmpfs, network share) only requires changing `vfs_server`'s routing table, not every client.

## How it builds on earlier phases

This phase builds on Core Servers and turns that foundation into Storage And VFS.

## Implementation outline

- Define a small file-oriented IPC protocol.
- Build `vfs_server` as a router instead of a full filesystem implementation.
- Add one filesystem backend with a narrow scope.
- Place a few test files in the boot media and verify they can be read.
- Keep caching and mutation out of the first milestone.
