## Why this phase matters

Programs can wait on multiple file descriptors simultaneously without busy-waiting. `select()`, an improved `poll()`, and `epoll` provide scalable I/O readiness notification. Non-blocking I/O (`O_NONBLOCK`) works for sockets, pipes, and PTYs. Event-driven servers become possible.

## How it builds on earlier phases

This phase builds on Expanded Memory and turns that foundation into I/O Multiplexing.

## Implementation outline

- Add `nonblock` flag to `FdEntry` and wire `fcntl(F_GETFL/F_SETFL)`.
- Implement `O_NONBLOCK` for pipes (simplest case).
- Implement `O_NONBLOCK` for sockets.
- Implement `O_NONBLOCK` for PTYs and TTY/stdin.
- Add per-FD wait queues to pipes, sockets, PTYs, and stdin.
- Rewrite `poll()` to use wait queues instead of busy-wait loop.
- Implement `select()` on top of the improved poll infrastructure.
- Implement `epoll_create1`, `epoll_ctl`, `epoll_wait`.
- Implement `accept4()`.
- Write test programs: echo server using poll, epoll-based multi-client server.
- Verify the telnet server works with improved poll (no busy-wait).
