## Why this phase matters

Add a minimal but real TCP/IP stack so the OS can send and receive packets over a network. A virtio-net NIC in QEMU is the hardware target. The end goal is a working `ping` and a simple TCP connection.

## How it works in m3OS

The roadmap entry defines the learning scope for this phase; a deeper implementation doc has not been matched yet.

## How it builds on earlier phases

This phase builds on Hardware Discovery and turns that foundation into Network.

## Implementation outline

- Use the PCI device list from Phase 15 to find the virtio-net device.
- Initialize the virtio device: feature negotiation, virtqueue setup.
- Implement the Ethernet and ARP layers.
- Implement IPv4 and ICMP; test with `ping 10.0.2.2` (QEMU default gateway).
- Implement UDP; test with a simple echo server.
- Implement TCP state machine: SYN, SYN-ACK, ACK, data, FIN.
- Expose socket syscalls through the POSIX compatibility layer, routing to net_server via IPC.
