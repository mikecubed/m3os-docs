## Why this phase matters

The OS has a service manager, system logging, and scheduled task execution. Services (sshd, telnetd, etc.) are managed by a unified init/service system that handles startup ordering, restart on failure, and clean shutdown. The OS behaves like a real server that can be administered remotely.

## Implementation outline

- Enhance init to read service definitions from `/etc/services.d/`.
- Implement dependency ordering (topological sort).
- Implement the `service` command.
- Implement service restart monitoring (SIGCHLD handler).
- Write `syslogd`: listen on `/dev/log` (or a named pipe), write to log files.
- Write the `logger` command.
- Write `crond`: parse crontab format, sleep until next job, execute.
- Write `crontab` command.
- Write `shutdown`, `reboot`, `hostname`, `date`, `who` utilities.
- Implement kernel reboot/shutdown syscalls.
- Test full lifecycle: boot → services start → cron runs → remote login → shutdown.
