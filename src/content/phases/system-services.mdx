## Why this phase matters

The OS has a service manager, system logging, and scheduled task execution. Services (sshd, telnetd, etc.) are managed by a unified init/service system that handles startup ordering, restart on failure, and clean shutdown. The OS behaves like a real server that can be administered remotely.

## How it works in m3OS

Phase 46 transforms m3OS from a system that hardcodes which daemons to run into one that reads service definitions from configuration files, starts them in dependency order, restarts them on failure, logs their output centrally, schedules recurring tasks, and shuts down cleanly. This is the layer that gives the later headless/reference release story a real operator-facing control plane.

## How it builds on earlier phases

This phase builds on Ports System and turns that foundation into System Services.

## Implementation outline

- Design the service definition file format (including `max_restart`) and create `/etc/services.d/` entries.
- Enhance init to parse service definitions and fill bidirectional dependency edges.
- Implement dependency-ordered startup (topological sort or iterative readiness check).
- Implement PID table mapping child PIDs to service entries for O(1) SIGCHLD dispatch.
- Implement `ServiceStatus` state machine enum (`NeverStarted -> Starting -> Running -> Stopping -> Stopped`).
- Implement SIGCHLD-based service exit detection with PID table lookup and automatic restart (capped by `max_restart`).
- Write the `service` command (start, stop, restart, status, list).
- Write `syslogd`: bind `/dev/log`, accept connections, write to log files.
- Write the `logger` command.
- Write `crond`: parse crontab format, compute next run time, sleep-and-execute loop.
- Write the `crontab` command.
- Implement `sys_reboot()` syscall and kernel shutdown path.
- Write `shutdown` and `reboot` commands.
- Write/update `hostname`, `who`, `w`, `last` utilities.
- Implement iterative reverse-dependency shutdown (find service with no unstopped dependents, stop it, repeat).
- Test full lifecycle: boot → services start → cron runs → remote login → shutdown.
