## Why this phase matters

Boot the kernel through UEFI, print useful output over serial, and halt cleanly. This is the first end-to-end proof that the workspace, toolchain, boot path, and panic path all work together.

## How it works in m3OS

The boot path has one job: get from a host-side Cargo command to running Rust code in a `no_std` kernel with enough early diagnostics to understand what happened.

## Implementation outline

- Set up the kernel crate and host-side `xtask`.
- Define the target and runner configuration used for OS builds.
- Initialize COM1 serial and expose simple print macros.
- Install a logger backend that writes through serial.
- Add a minimal `kernel_main` entry point and `hlt` loop.
- Add a panic handler that emits enough context to debug early failures.
