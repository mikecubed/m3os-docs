## Why this phase matters

The kernel heap allocator is robust, efficient, and recoverable. Out-of-memory conditions no longer panic the kernel — the allocator grows on demand and retries failed allocations. Buddy allocation, working `munmap()`, userspace-heap coalescing, and slab-cache infrastructure all land. Broad migration of hot kernel object families onto slab caches remains later work.

## Implementation outline

- Implement OOM retry wrapper around the global allocator.
- Verify: allocation-heavy workloads no longer panic (heap grows and retries).
- Implement buddy allocator for page-granularity allocations.
- Implement slab allocator caches for Task, FD, endpoint, pipe, and socket-sized objects, and validate them in direct cache tests.
- Implement working `munmap()` with frame reclamation and TLB invalidation.
- Add free-block coalescing to userspace `BrkAllocator`.
- Add `heap_stats()` and verify with a stress-test program.
- Run existing test suite to verify no regressions.
