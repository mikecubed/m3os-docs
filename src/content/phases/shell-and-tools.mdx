## Why this phase matters

Turn the minimal shell from Phase 9 into a genuinely useful interactive environment: pipes, I/O redirection, job control, environment variables, and a small set of core utilities that exercise the filesystem and process model.

## How it works in m3OS

The roadmap entry defines the learning scope for this phase; a deeper implementation doc has not been matched yet.

## How it builds on earlier phases

This phase builds on Writable FS and turns that foundation into Shell And Tools.

## Implementation outline

- Add file descriptor table per process: integers mapping to open file descriptions.
- Implement `pipe` syscall: allocate a kernel pipe buffer, return two fds.
- Implement `dup2` syscall: redirect one fd to another before `execve`.
- Add signal infrastructure: `SIGINT`, `SIGTSTP`, `SIGCONT`, `SIGCHLD` at minimum.
- Wire `Ctrl-C` and `Ctrl-Z` in the keyboard server to deliver signals to the foreground process group.
- Implement process groups so the shell can address foreground and background jobs.
- Extend the shell parser to handle `|`, `>`, `<`, `>>`, `&`, and variable expansion.
- Compile each utility as a standalone static ELF and add to the disk image.
