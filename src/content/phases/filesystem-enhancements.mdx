## Why this phase matters

The filesystem supports symlinks, hard links, proper permission enforcement, a `/proc` filesystem for process introspection, and basic device nodes (`/dev/null`, `/dev/zero`, `/dev/urandom`). These are the filesystem primitives that Unix tools and build systems assume exist.

## How it builds on earlier phases

This phase builds on I/O Multiplexing and turns that foundation into Filesystem Enhancements.

## Implementation outline

- Implement symlinks in tmpfs (create, readlink, path resolution with follow).
- Implement symlinks in ext2.
- Add symlink loop detection (max 40 hops, return `ELOOP`).
- Implement hard links in ext2 (link count management).
- Add permission checking to `open()`, `exec()`, directory operations.
- Implement `umask()` syscall with per-process mask.
- Implement `/proc` filesystem (start with `/proc/<pid>/status` and `/proc/meminfo`).
- Add `/dev/null`, `/dev/zero`, `/dev/urandom`, `/dev/full` device nodes.
- Implement `statfs`/`fstatfs`.
- Write `ln` utility for creating links.
- Test: `ln -s /bin/sh0 /tmp/mysh && /tmp/mysh` works via symlink.
- Test: permission enforcement blocks unauthorized access.
