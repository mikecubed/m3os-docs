## Why this phase matters

Replace the hardcoded, single-core interrupt model with proper hardware discovery via ACPI and PCI, and switch from the legacy 8259 PIC to the APIC. This unlocks multi-core support and arbitrary device discovery in later phases.

## How it works in m3OS

The roadmap entry defines the learning scope for this phase; a deeper implementation doc has not been matched yet.

## How it builds on earlier phases

This phase builds on Shell And Tools and turns that foundation into Hardware Discovery.

## Implementation outline

- Parse the RSDP address from `BootInfo` (UEFI already locates it).
- Walk the RSDT/XSDT and collect pointers to each SDT.
- Parse the MADT and record the Local APIC base address and all I/O APIC descriptors.
- Identity-map the Local APIC MMIO page and write the initialization sequence.
- Program the I/O APIC redirection table for the IRQs the kernel currently handles.
- Disable the 8259 PIC.
- Switch the scheduler's timer source from the PIT to the LAPIC timer.
- Implement PCI config space reads (port I/O: address port 0xCF8, data port 0xCFC).
- Enumerate all PCI functions and store the device list in a static kernel array.
