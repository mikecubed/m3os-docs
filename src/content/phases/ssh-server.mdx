## Why this phase matters

An SSH server runs inside the OS, providing encrypted remote shell access. Users can connect from any standard SSH client (`ssh user@host`) with password or public key authentication. This replaces the telnet server for untrusted networks and is a major milestone: the OS is now a secure, networked, multi-user system.

## How it builds on earlier phases

This phase builds on Async Executor and turns that foundation into SSH Server.

## Implementation outline

- Evaluate sunset crate: add dependency, verify it compiles for x86_64-m3os target.
- Create `userspace/sshd/` crate with basic TCP accept loop.
- Generate Ed25519 host key on first boot, store at `/etc/ssh/ssh_host_ed25519_key`.
- Implement sunset adapter: feed TCP bytes into sunset, write output back to socket.
- Implement password authentication callback against `/etc/shadow`.
- Implement session channel handling: PTY allocation, shell exec, data relay.
- Implement public key authentication with `~/.ssh/authorized_keys`.
- (Deferred) Window-change and signal forwarding.
- Add sshd to init's startup sequence and initrd.
- Test from host with `ssh -p 2222 user@localhost`.
- Test multiple simultaneous SSH sessions.
- Verify encryption by inspecting traffic.
