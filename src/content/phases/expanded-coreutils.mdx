## Why this phase matters

The OS ships with a comprehensive set of Unix utilities sufficient for daily development work. Moving beyond the minimal set from Phase 14, this phase adds text processing, file inspection, and system administration tools that developers expect.

## How it works in m3OS

The roadmap entry defines the learning scope for this phase; a deeper implementation doc has not been matched yet.

## How it builds on earlier phases

This phase builds on Threading Primitives and turns that foundation into Expanded Coreutils.

## Implementation outline

- Set up sbase cross-compilation with musl on the host.
- Port text processing tools first (head, tail, sort, uniq — most immediately useful).
- Port file tools (find, xargs, tee).
- Build the system tools around existing procfs and syscall support: `ps`, `free`, `mount`, and `uptime` read the Phase 38 `/proc` files, while new kernel work stays limited to the `dmesg` ring buffer and `umount`.
- Port diff and patch — essential for development workflows.
- Port or write a pager (less/more).
- Port remaining tools.
- Test each tool with shell pipelines to verify composability.
