## Why this phase matters

m3OS gains an explicit, enforceable architecture contract for what belongs in ring 0, what is transitional, and what must move to userspace over time. The phase also turns the giant syscall surface into a structure that can be unwound safely in later phases.

## How it works in m3OS

Phase 49 makes the kernel/userspace boundary explicit and enforceable. Before this phase, m3OS described itself as a microkernel but lacked a formal contract for what belongs in ring 0, what is transitional, and what must eventually move to userspace. This phase closes that gap by decomposing the syscall surface, classifying every kernel subsystem by long-term ownership, and adopting a userspace-first rule for new policy-heavy code.

## How it builds on earlier phases

This phase builds on Security Foundation and turns that foundation into Architectural Declaration.

## Implementation outline

- Inventory the current kernel, syscall, and service boundaries against the documented microkernel ideal.
- Create a keep/move/transition matrix and link it from the architecture and roadmap docs.
- Split `syscall.rs` into subsystem-oriented modules or an equivalent staged decomposition.
- Tag major modules and service paths by long-term ownership.
- Document the userspace-first rule for future policy-heavy work.
- Update roadmap dependencies and contributor-facing docs to reflect the declared architecture.
- Validate that the refactor still builds cleanly and that the docs match the code layout.
